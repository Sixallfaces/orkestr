# Простое руководство по плагину оркестрации

Это краткое объяснение на человеческом языке, чтобы быстро понять, что делает плагин и как им пользоваться, даже если вы не программист.

## Что это за плагин
- **Для чего нужен:** помогает просить ИИ выполнять сложные цепочки задач шаг за шагом или параллельно.
- **Как думать:** как «сценарий действий», который вы задаёте словами или короткими блок-схемами. Плагин сам создаёт временные скрипты (Python/Node.js), вызывает нужных «агентов» и показывает прогресс.

## Что надо иметь
1. Установленный Claude Code.
2. Интернет-доступ, чтобы установить плагин из маркетплейса.

## Как установить
1. Откройте Claude Code и добавьте маркетплейс плагинов:
   ```bash
   /plugin marketplace add mbruhler/claude-orchestration
   ```
2. Поставьте плагин:
   ```bash
   /plugin install orchestration@mbruhler
   ```
   Можно открыть меню `/plugin`, выбрать **Browse Plugins**, найти **orchestration** и нажать **Install**.
3. Проверьте, что появился список команд `/help`.

## Как им пользоваться без кода
### 1) Спросить на обычном языке
Пример: «Сделай воркфлоу: найди 10 постов про стартапы на Reddit, выпиши конкурентов и сделай табличку рейтингов». 
Плагин сам предложит вопросы, создаст временные скрипты и покажет результат таблицей.

### 2) Использовать готовые шаблоны
Пример: «Используй шаблон TDD» — плагин построит последовательность шагов под тестирование.

### 3) Набросать простую блок-схему
```flow
[Собрать ссылки || Скачать тексты] ->
Проанализировать:"Главные темы":темы ->
@review:"Всё ок?" ->
Сделать отчёт
```
- `[]` и `||` означают параллельные шаги.
- `@review` добавляет ручную остановку: плагин спросит «продолжать?»
- `"...":переменная` сохраняет результат шага, чтобы использовать дальше.

## Важные фишки
- **Авто-скрипты:** когда нужно сходить в сеть, обработать данные или вызвать API, плагин сам пишет и запускает временные скрипты.
- **Встроенные агенты:** есть роли вроде "general-purpose" (универсальный), "code" (работа с кодом), "@review" (ручная проверка). Их можно комбинировать.
- **Видимость прогресса:** плагин рисует текстовые карточки с текущим статусом, поэтому понятно, на каком шаге задача.

## Мини-курс на 5 минут
1. Установите плагин.
2. Запустите `/orchestration:menu` и выберите быстрый старт или импорт своих агентов (`/orchestration:init`).
3. Сформулируйте цель одной фразой или вставьте короткий блок `flow` (см. выше).
4. Когда появятся контрольные точки `@review`, ответьте «продолжать» или скорректируйте запрос.
5. Сохраните понравившиеся сценарии как свои шаблоны и повторяйте при следующих задачах.

## Если что-то идёт не так
- Нет команды `/orchestration:*`? Проверьте, установлен ли плагин и перезапустите команду `/plugin`.
- Шаг завис? Посмотрите текстовый прогресс: если плагин ждёт ответа на `@review`, нужно подтвердить.
- Ошибка в сети? Попробуйте ещё раз — временные скрипты пересоздаются автоматически.

Готово! Теперь можно строить цепочки действий простыми словами и не писать код.
